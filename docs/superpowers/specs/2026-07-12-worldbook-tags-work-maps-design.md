# 世界书标签与打工地图设计

## 目标

修复世界书首页头图缺失和标题被灵动岛遮挡的问题，并将世界标签升级为最多三个标签。打工 APP 根据当前世界的标签提供可切换的独立地图。

## 世界书首页

- 头图使用独立的绝对定位背景层，不依赖内容容器的安全区内边距。
- 背景层覆盖屏幕顶部和 iOS 状态栏安全区，保持 `object-fit: cover`。
- 返回、搜索、新建按钮与标题使用单独的前景层，起点位于 `env(safe-area-inset-top)` 下方。
- 删除会把整个前景层拉回状态栏的负边距或重复安全区补偿。
- 首页世界条目仅显示世界名称和已选具体标签，不显示标签总分类。

## 打工地图

- 25 个标签分别映射到 25 个独立主题。
- 每个主题有独立底图、五个地点、工作文案、点击区域和建筑轮廓。
- 地图构图不复用固定的左右交替模板。地图采用环形、中心辐射、纵深街巷、对角路径、群岛散点、上下分层、星轨节点等不同空间结构。
- 点击区域必须贴合各自建筑或区域外观，不使用统一矩形视觉框。
- 当前世界有多个标签时，打工页顶部显示最多三个紧凑标签按钮。默认选择第一个标签，点击标签立即切换地图与对应工作。
- 工作进行期间仍可切换标签、查看其他地图和工作点，但不能开始第二份工作。
- 现实模式继续使用现代地图，不显示世界标签切换器。
- API 生成工作时只允许使用当前标签地图定义的五个地点。

## 地图素材策略

每个标签拥有一张真实生成的完整底图。底图覆盖全屏并延伸至底部安全区，文字与控件悬浮其上。建筑轮廓资源与底图坐标一一对应，不通过简单调色或裁剪复用其他地图。

## 验证

- 单元测试验证标签顺序、最多三个标签、旧数据迁移、标签到主题映射及每个主题的不同点位。
- 手机视口验证世界书头图覆盖状态栏、前景避开灵动岛、标签不溢出。
- 手机视口验证世界切换、标签地图切换、工作中只读切换和工作开始限制。
- 构建后同步到 `docs/`，递增版本号并检查 GitHub Pages 线上版本。
